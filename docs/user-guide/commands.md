## 命令概述

所有斜杠命令以 `/` 开头，在输入框中输入即可执行。命令按来源分为三类：

| 来源 | 优先级 | 位置 |
|------|--------|------|
| **项目级命令** | 最高 | `.orangecoding/commands/*.md` |
| **用户全局命令** | 中 | `~/.orangecoding/commands/*.md` |
| **内置命令** | 最低 | 程序内置 |

相同名称的命令，高优先级覆盖低优先级。

## 基础命令

### `/help`

显示帮助信息和可用命令列表。

**别名：** `/hotkeys`

**用法：**

```
/help
```

**说明：** 展示所有可用的斜杠命令、快捷键以及基本使用说明。

---

### `/clear`

清除当前屏幕显示。

**用法：**

```
/clear
```

**说明：** 仅清除终端显示内容，不影响会话历史。会话上下文保持不变。

---

### `/compact`

手动压缩上下文，减少 Token 消耗。

**用法：**

```
/compact
/compact [聚焦主题]
```

**参数：**

| 参数 | 必须 | 说明 |
|------|------|------|
| `聚焦主题` | 否 | 压缩时重点保留的内容主题 |

**示例：**

```
/compact                    # 通用压缩
/compact API 性能优化        # 压缩时重点保留 API 性能相关的上下文
/compact 用户认证模块         # 压缩时重点保留认证相关上下文
```

**说明：**
- 默认在上下文超过 100K Token 时自动触发
- 保留最近 10 条消息不被压缩
- 指定聚焦主题可以确保重要信息不在压缩中丢失
- 压缩会生成摘要替代详细历史

---

### `/model`

打开模型选择器，切换当前 Agent 使用的 AI 模型。

**别名：** `/models`

**用法：**

```
/model
/models
```

**说明：** 打开交互式模型选择器，列出所有可用的 AI 模型。选择后立即生效，影响当前 Agent 后续的所有对话。

**可选模型示例：**
- `claude-opus-4-6`、`claude-sonnet-4-6`、`claude-haiku`
- `gpt-5.4`、`gpt-5.4-mini`
- `deepseek-chat`、`deepseek-coder`
- `qwen-turbo`、`qwen-plus`

---

### `/status`

显示当前系统状态。

**用法：**

```
/status
```

**说明：** 显示当前 Agent、模型、会话信息和系统运行状态。

---

### `/usage`

显示当前会话的 Token 使用统计。

**用法：**

```
/usage
```

**输出信息：**
- 输入 Token 数量
- 输出 Token 数量
- 缓存读取 Token 数量
- 缓存写入 Token 数量
- 总计费用估算

---

### `/exit`

退出 OrangeCoding。

**别名：** `/quit`

**用法：**

```
/exit
/quit
```

## 工作流命令

### `/init-deep`

深度初始化：扫描项目结构，创建状态文件，初始化 Agent 工作状态。

**用法：**

```
/init-deep
/init-deep [路径]
```

**参数：**

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| `路径` | 否 | `.`（当前目录） | 要扫描的项目路径 |

**示例：**

```
/init-deep                  # 扫描当前目录
/init-deep src/core         # 仅扫描 src/core 子目录
```

**执行过程：**
1. 递归扫描项目文件结构
2. 创建 `.sisyphus/boulder.json` 状态文件
3. 初始化 Agent 工作上下文
4. 分析项目技术栈和依赖

**建议：** 在首次使用 OrangeCoding 打开新项目时执行此命令。

---

### `/ralph-loop`

Ralph 持续改进循环：自动执行 plan → implement → review → refine 的迭代流程。

**用法：**

```
/ralph-loop
/ralph-loop [聚焦区域]
```

**参数：**

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| `聚焦区域` | 否 | `全局` | 要重点改进的代码区域或模块 |

**示例：**

```
/ralph-loop                  # 全局持续改进
/ralph-loop API layer        # 聚焦 API 层的改进
/ralph-loop 错误处理          # 聚焦错误处理的改进
```

**执行流程：**

```
┌─────────┐    ┌──────────────┐    ┌──────────┐    ┌──────────┐
│  Plan   │ →  │ Implement    │ →  │  Review  │ →  │  Refine  │
│  规划    │    │  实现         │    │  审查     │    │  精炼     │
└─────────┘    └──────────────┘    └──────────┘    └──────────┘
      ↑                                                  │
      └──────────────────────────────────────────────────┘
                          持续循环
```

**停止方式：** 使用 `/stop-continuation` 命令终止循环。

---

### `/ulw-loop`

UltraWork 全自动模式：启动完全自动化的开发流程，OrangeCoding 将自主完成从分析到验证的全部步骤。

**用法：**

```
/ulw-loop
/ulw-loop [配置]
```

**参数：**

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| `配置` | 否 | `默认配置` | 自定义 UltraWork 配置名称 |

**示例：**

```
/ulw-loop                    # 使用默认配置启动
/ulw-loop custom-config      # 使用自定义配置
```

**六阶段工作流程：**

| 阶段 | 说明 |
|------|------|
| 1. **Scanning**（扫描） | 扫描项目结构，识别关键文件 |
| 2. **Exploring**（探索） | 深入分析代码，理解架构 |
| 3. **Planning**（规划） | 制定实现计划 |
| 4. **Executing**（执行） | 自动执行代码修改 |
| 5. **Verifying**（验证） | 运行测试，验证正确性 |
| 6. **Done**（完成） | 输出总结报告 |

**默认配置参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_parallel_agents` | 3 | 最大并行 Agent 数 |
| `enable_self_correction` | `true` | 启用自动纠错 |
| `enable_deep_research` | `false` | 启用深度研究 |

**替代触发方式：** 在对话中使用 `ultrawork` 或 `ulw` 关键词也可触发：

```
> ulw 实现用户注册和登录功能
> ultrawork 重构数据库层
```

---

### `/refactor`

重构助手：分析目标代码并提出重构建议。

**用法：**

```
/refactor
/refactor [目标]
```

**参数：**

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| `目标` | 否 | 当前上下文 | 要重构的文件或模块名 |

**示例：**

```
/refactor                    # 基于当前上下文分析重构机会
/refactor main.rs            # 分析 main.rs 的重构建议
/refactor src/auth/          # 分析认证模块的重构建议
```

**输出内容：**
- 代码异味（Code Smells）识别
- 重构方案建议
- 影响范围评估
- 风险等级标注

---

### `/start-work`

开始工作会话：创建新的工作会话并初始化 Boulder 状态。

**用法：**

```
/start-work
/start-work [任务描述]
```

**参数：**

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| `任务描述` | 否 | 无 | 本次工作的目标描述 |

**示例：**

```
/start-work                              # 开始新工作会话
/start-work 实现用户认证模块               # 开始认证模块的开发
/start-work fix: 修复订单计算精度问题       # 开始修复特定 bug
```

**执行过程：**
1. 创建新的 Boulder 工作会话
2. 初始化 `.sisyphus/boulder.json`
3. 记录任务描述和开始时间
4. 设置 Agent 工作状态

---

### `/handoff`

任务交接：将当前任务从当前 Agent 转交给指定的 Agent。

**用法：**

```
/handoff [agent名称]
```

**参数：**

| 参数 | 必须 | 说明 |
|------|------|------|
| `agent名称` | 是 | 目标 Agent 的名称 |

**示例：**

```
/handoff hephaestus         # 将任务交给 Hephaestus（深度工作）
/handoff oracle             # 将任务交给 Oracle（架构分析）
/handoff prometheus         # 将任务交给 Prometheus（战略规划）
```

**说明：**
- 交接时会传递当前上下文和工作状态
- 通过 `chengcoding-mesh` 消息总线实现 Agent 间通信
- 如果不指定 Agent 名称，会提示选择

---

### `/stop-continuation`

停止继续：终止当前正在运行的自动循环。

**用法：**

```
/stop-continuation
```

**说明：** 用于终止 `/ralph-loop`、`/ulw-loop` 等自动循环命令。

## 会话管理命令

### `/new`

开始一个新的空白会话。

**用法：**

```
/new
```

**说明：** 创建全新会话，上下文从零开始。之前的会话不会被删除，可以通过 `/resume` 恢复。

---

### `/resume`

打开会话选择器，恢复之前的会话。

**用法：**

```
/resume
```

**说明：** 显示交互式会话列表，选择要恢复的历史会话。

---

### `/session`

显示当前会话的详细信息。

**用法：**

```
/session
```

**输出内容：**
- 会话 ID
- 创建时间
- 消息数量
- Token 使用量
- 当前 Agent 和模型

---

### `/export`

将当前会话导出为 HTML 文件。

**用法：**

```
/export
/export [路径]
```

**参数：**

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| `路径` | 否 | 自动生成 | 导出文件的保存路径 |

**示例：**

```
/export                          # 导出到默认位置
/export ./session-export.html    # 导出到指定路径
```

---

### `/tree`

以树形结构展示会话的消息分支。

**用法：**

```
/tree
```

**说明：** 会话支持分支功能，`/tree` 可视化展示消息的树形结构和分支点。

---

### `/branch`

打开分支选择器。

**用法：**

```
/branch
```

**说明：** 当会话有多个分支时，可以切换到不同的分支。

---

### `/fork`

从当前消息创建分支。

**用法：**

```
/fork
```

**说明：** 在当前消息处创建一个新分支，之后的对话将沿新分支发展，原分支保持不变。

---

### `/copy`

复制最后一条 AI 回复到剪贴板。

**用法：**

```
/copy
```

---

## 自定义命令

### 创建自定义命令

在 `.orangecoding/commands/` 或 `~/.orangecoding/commands/` 目录下创建 Markdown 文件即可注册自定义命令。

**文件格式：**

```markdown
---
description: 命令描述
---
命令模板内容，可使用 $1, $2, $ARGUMENTS 等占位符
```

**文件名** 即为命令名称（不含 `.md` 后缀）。

### 示例：自定义代码审查命令

创建文件 `.orangecoding/commands/review.md`：

```markdown
---
description: 对指定文件执行代码审查
---
请对以下文件进行详细的代码审查：$1

审查要点：
1. 代码逻辑正确性
2. 错误处理完备性
3. 性能考虑
4. 安全隐患
5. 代码风格一致性

请给出具体的改进建议。
```

使用方式：

```
/review src/auth/login.rs
```

### 示例：自定义测试命令

创建文件 `.orangecoding/commands/test-module.md`：

```markdown
---
description: 运行指定模块的测试并分析结果
---
请执行以下操作：
1. 运行模块 $1 的所有测试: cargo test -p $1
2. 分析测试结果
3. 如有失败的测试，给出修复建议
```

使用方式：

```
/test-module chengcoding-config
```

### 占位符说明

| 占位符 | 说明 |
|--------|------|
| `$1` | 第一个参数 |
| `$2` | 第二个参数 |
| `$@` | 所有参数 |
| `$ARGUMENTS` | 所有参数（同 `$@`） |
